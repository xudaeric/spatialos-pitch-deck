The Shift

The room becomes the computer.

过去四十年，计算中心从 PC 到手机、云和 AI agent。物理空间仍然停在上一代：设备联网了，但空间本身不会记忆、协调或执行。

PastDevices respond to commands.
NowAI understands intent, but cannot reliably act in the room.
NextSpace coordinates objects, energy, permissions, memory, and motion.
Device-centric手机、App、遥控器、语音助手各自控制一个设备。
Agent-centricAI 可以理解目标，却缺少可验证的物理执行接口。
Space-centric房间维护状态、分配资源、调度节点，并留下可审计记录。

Planner explanation

Type-2 Robot Planner

When you ask for a table, nine identical cabinets become the surface.

你不需要知道算法。你只说“我要一张多大的桌面”，系统让 9 个 60cm x 60cm x 50cm 柜子移动、升降或展开桌面机构，决定它们的位置、排列和变形方式。

中文解释

第二类机器人像带可变形桌面的移动柜子。先安全移动到一起，再让桌面通过滑移、翻折、升降、折纸翼片等方式变大。

Plain English

Each robot keeps the same cabinet body. SpatialOS first coordinates traffic, then runs the selected tabletop mechanism.

安全边界

移动层和变形层分开验证：移动时只看 60cm 底座、障碍物和人体安全区；到位锁定后才开始桌面机构。

Why Rush Hour matters

Rush Hour shows that moving many rectangles can become very hard. So we split “where should they end up” from “how should they move there”.

User: “I need a 2.4m by 3.0m tabletop.”
  1. Keep all 9 cabinet bases fixed at 60cm square.
  2. Split the requested rectangle into a 3 x 3 surface grid.
  3. Plan traffic with obstacle and human keep-out zones.
  4. Negotiate slot swaps when a route is blocked.
  5. After bases lock, animate the tabletop mechanism.

Planner pseudocode and code

Pseudocode + Reference Code

The planner is a bounded search, not magic.

伪代码给设计师和机器人团队读；Python 参考实现已经在仓库中。前面 8 个动画页来自变形桌调研：几何滑移、伸缩蝶叶、翻盖、控制台翻折、中心叶、电动单轴、升降展开、折纸翼片。

伪代码 / Pseudocode
PLAN_TABLE_COMPOSITION(request, robots):
  candidates = flat + available + same height
  queue = [empty layout]
  while time remains:
    partial = best unfinished layout
    if target is covered:
      score and keep best plan
    anchor = first uncovered point
    for each robot rectangle:
      try 0° and 90° rotation
      reject overlap, collision lane, gap, overhang
      push improved layout
  return best feasible plan or NO_SOLUTION
Python reference
plan = plan_table_composition(
  TableRequest(length_m, width_m, tolerance_m=0.0),
  robots
)

for p in plan.placements:
  send_goal(
    robot_id=p.robot_id,
    center=(p.center_x, p.center_y),
    yaw=p.yaw,
    rectangle=(p.rect_length, p.rect_width)
  )
